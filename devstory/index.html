WHY

왜 만들었나

경제 뉴스 앱을 열어놓고도 "이게 무슨 뜻이야?" 하고 묻는 가족이 있었습니다. 기사는 쏟아지는데, 뭐가 중요한 뉴스인지는 전혀 모르겠다는 거죠.

💡

매일 아침, AI가 4줄로 요약해주면 어떨까?

이 생각 하나로 시작했습니다. 그런데 저는 개발자가 아니었어요. 전략기획 일을 했고, 코드는 HTML 태그 조금 아는 수준이 전부였습니다.

만들고 싶은 건 명확한데 만들 줄은 모르는 상태. 그때 클로드 코드를 처음 켰습니다.

WHAT IS CLAUDE CODE

클로드 코드가 뭔가요?

한 줄로 말하면 — 터미널에서 한국어로 대화하는 AI 개발자입니다. 제가 코드를 쓰는 게 아니라, 제가 말하고 클로드가 씁니다.

💬

대화가 전부예요

"로딩 화면에 심전도 애니메이션 넣어줘" 하고 한국어로 말하면, 클로드가 실제 CSS 파일을 읽고 고치고 저장합니다. 제가 코드 문법을 몰라도 작동합니다.

🗂

제 컴퓨터를 직접 봐요

웹 채팅창(claude.ai)과 달리, 클로드 코드는 실제 파일 시스템에 접근합니다. 코드 복사 붙여넣기가 필요 없어요. "그 파일 고쳐줘" 하면 알아서 찾고 고쳐놓습니다.

⚡

배포도 같이 해줘요

"이제 배포해줘" 하면 git commit·push까지 알아서 실행합니다. 10초 후 전 세계에서 바뀐 앱이 열립니다. 터미널 1~2개 명령어만 알면 끝입니다.

📌

Cursor · Windsurf · GitHub Copilot과 뭐가 다른가요?

이들은 코드 에디터 안에서 AI가 보조하는 형태입니다. 클로드 코드는 터미널 자체가 AI라서 에디터를 굳이 안 열어도 됩니다. "README 업데이트해줘" 하면 저 대신 파일을 열고 쓰고 저장해놓죠. 코딩하는 느낌보다는 비서에게 지시하는 느낌에 가깝습니다.

THE BUILD

실제로 어떻게 만들었나 (대화 예시와 과정)

제가 클로드에게 어떻게 지시하고, 클로드가 어떻게 답했는지 — 실제 대화 스크린샷과 빌드 흐름을 공개합니다.

📸 대화 스크린샷 · 실제 요청과 응답 예시 · 단계별 빌드 과정

STRUGGLES

겪은 시행착오 (이걸 알고 시작했으면 좋았을 것들)

앱을 만들면서 실제로 부딪혔던 문제들입니다. 클로드 코드가 만능이었다면 이런 일은 없었겠죠.

문제 1 로딩 시간이 너무 길다

AI가 뉴스를 분석하려면 최소 10~30초가 걸립니다. 앱을 열었는데 아무것도 안 보이면 사용자는 그냥 떠나버리더라고요.

🔧 처음엔 이런 걸 시도했어요

① 순차 표시: 1차 브리핑이 끝나면 즉시 보여주고, 나머지는 뒤에서 처리. 기다리는 체감을 줄였습니다.
② 프롬프트 단축: AI 지시문을 최대한 짧게 다듬어 처리 시간을 줄였습니다.
③ 로딩 UX: 스피너 대신 ECG 심전도 애니메이션 + 경제 퀴즈로 대기 체감 시간 축소.

✅ 진짜 해결책 — Firebase 사전 생성

"사용자가 요청할 때 AI 호출" 방식 자체를 버렸습니다. 매일 07:00 KST에 서버가 미리 브리핑을 만들어 Firebase에 저장하고, 사용자는 그걸 꺼내 보기만 합니다. AI 호출 0초, 로딩 ~0.5초.

문제 2 AI가 뉴스를 거꾸로 요약한다

할루시네이션이라고, AI가 사실이 아닌 걸 사실처럼 지어내는 현상이 있습니다. 비용을 아끼려고 저렴한 모델(Haiku)을 썼더니 "A기업 주가 상승"이란 뉴스를 "A기업 주가 하락"으로 요약하는 일이 실제로 생겼어요. 경제 뉴스에서 방향이 틀리면 그냥 쓰레기죠.

해결: Haiku → Sonnet으로 교체 (비용은 올라갔지만 정확도 크게 개선). 프롬프트에도 "제공된 뉴스에만 근거하라"는 지시를 박아놨습니다.

문제 3 개발할 때마다 AI 비용이 나간다

글자 색 하나 바꾸는 UI 작업인데, 테스트하려고 앱을 실행하면 AI가 호출돼서 비용이 발생합니다. 실제로 하루 $2.38이 예상 외로 찍힌 적이 있어요.

해결: DEV MODE 도입. URL 뒤에 ?dev=true를 붙이면 AI를 호출하지 않고 미리 만들어둔 가짜 데이터로 작동합니다. 화면은 실제와 똑같지만 비용은 0원입니다.

더 많은 시행착오(크론 데드락, PWA 엣지 스와이프, API 비용 유출 등)는 다음 업데이트에서 풀어쓸 예정입니다 (Phase B)

FOR BEGINNERS

클로드 코드 시작하시는 분께

제가 6개월 써보면서, 처음에 알았으면 좋았을 것 3가지입니다. 기술 팁이 아니라 태도에 가까워요.

💰

1. 비용을 각오하세요

클로드 코드 자체는 Pro/Max 구독($20~$100/월)으로 쓸 수 있지만, 앱에서 AI API를 호출하면 그건 또 별도 비용입니다. 저는 테스트 중 하루에 $2.38이 찍힌 적이 있어요.

반드시 DEV MODE를 먼저 만드세요. 더미 데이터로 UI를 확인할 수 있는 스위치 하나 — 이게 있고 없고가 월 비용을 10배 가른다고 보시면 됩니다.

📋

2. 합의 먼저, 실행은 그 다음

똑똑한 AI일수록 자기가 옳다고 판단하면 바로 실행해버립니다. 특히 상위 모델(Opus)은 자기가 최선이라 생각하는 방향으로 코드를 쏟아내요.

저는 "모든 변경은 먼저 설명하고, 합의한 뒤에 진행할 것"을 CLAUDE.md 규칙으로 박아놨습니다. 이 한 줄이 많은 사고를 막아줍니다.

🔄

3. Git Push가 생명줄이에요

AI가 쏟아낸 코드는 언제든 뒤집을 수 있어야 합니다. 작업 단위마다 git commit && git push — 그리고 Vercel 같은 배포 서비스와 연결해두면 배포까지 자동입니다.

뒤로 돌아갈 수 있다는 안전망이 있어야 과감하게 실험할 수 있어요. 실수해도 git reset 한 번이면 끝입니다.

🎯

그리고 가장 중요한 것

클로드 코드는 "코딩을 배워주는 도구"가 아니라 "내 아이디어를 실제 서비스로 만들어주는 도구"입니다. 그래서 가장 많이 써야 할 근육은 코딩 실력이 아니라 생각의 구조예요. 무엇을 만들고 싶은지, 왜 그게 필요한지, 어떤 순서로 만들 건지 — 여기서 결과의 90%가 갈립니다. 좋은 지시가 좋은 결과를 만듭니다.
