WHY

왜 경제숲을 만들었는가?

🧩

파편화된 정보

경제 뉴스가 수십 개 매체에 흩어져 있습니다. 중요한 뉴스를 찾으려면 여러 사이트를 돌아다녀야 하고, 정작 핵심이 뭔지 파악하기 어렵습니다.

🎯

알고리즘의 편향

플랫폼 알고리즘은 내 관심사만 보여줍니다. 내가 좋아하는 것만 계속 추천하니, 정작 객관적으로 중요한 정보는 놓치게 됩니다.

⏱

시간의 부족

바쁜 일상 속에서 경제 뉴스를 하나하나 읽을 시간이 없습니다. 핵심만 빠르게 파악할 수 있는 방법이 필요했습니다.

💡

그래서 떠오른 아이디어

"AI 모델을 활용해서 뉴스를 자동으로 수집하고 분석하면, 오래 인터넷을 떠돌지 않더라도 1분 이내에 오늘의 주요 정보를 파악할 수 있지 않을까?"

이 생각이 경제숲의 출발점이었습니다. 누구나 가지고 있는 이 니즈를, 바이브코딩으로 직접 해결해보기로 했습니다.

경제숲이 해결하는 것

❌ 기존 방식

여러 뉴스 사이트를 돌아다니며 30분 이상 소요. 알고리즘이 추천하는 편향된 정보만 소비. 핵심이 뭔지 판단은 본인 몫.

✅ 경제숲

앱을 열면 1분 이내에 핵심 파악 완료. AI가 객관적으로 중요한 뉴스를 선별. 배경·시장·전망까지 한눈에.

DATA FLOW

앱을 열면 어떤 일이 벌어지나?

사용자 눈에는 3줄 브리핑이 보이지만, 뒤에서는 이런 과정이 진행됩니다.

1

뉴스 수집 — Naver API

키워드별로 Naver 뉴스 API에 요청을 보냅니다. 18개 헤드라인을 확보하되, 3개씩 배치로 250ms 간격을 두어 API 429(과다 요청) 에러를 방지합니다. 중복과 스팸 키워드를 필터링합니다.

2

1차 AI 호출 — briefing.js

수집된 헤드라인 + 설명(앞 100자)을 Claude Sonnet에 전달합니다. AI가 SUMMARY 3줄 + FOOTNOTES(용어 각주)를 생성하면 즉시 화면에 렌더링합니다. 사용자는 이 단계에서 바로 브리핑을 볼 수 있습니다.

3

2차 AI 호출 — insight.js (백그라운드)

1차 결과를 기반으로 배경(Why) / 시장(So What) / 주목(Next Move) 쓰리포인트 분석을 생성합니다. 이 호출은 백그라운드에서 진행되어 사용자 체감 속도에 영향을 주지 않습니다.

4

캐시 저장 — localStorage

결과를 브라우저 localStorage에 저장합니다. 다음 스케줄 시간까지 캐시를 재사용하여 불필요한 API 호출을 차단합니다.

⚡

핵심 설계 원칙

1차 호출이 끝나면 즉시 화면에 보여주고, 2차는 백그라운드에서 처리합니다. 체감 속도 최우선이 경제숲의 로딩 전략입니다.

📦

## 실제 배포 과정 — 이 2줄이 전부입니다 cd /c/Users/dodoh/OneDrive/Desktop/Project101/ecobrief git add -A && git commit -m "로딩 UX 개선 (v97)" && git push ## → 10초 후 전 세계에 배포 완료

Vercel — 서버 없이 서비스 운영

🔧 서버리스 함수

🚀 자동 배포

GitHub Push → 10초 내 전 세계 CDN 배포. 도메인 설정, SSL 인증서 작업이 전혀 없습니다.

⏱ 60초 타임아웃

AI API 호출은 시간이 오래 걸릴 수 있어 vercel.json에서 함수별 타임아웃을 60초로 설정했습니다.

🔐 환경 변수

NAVER_CLIENT_ID, ANTHROPIC_API_KEY 같은 민감한 키를 Vercel 대시보드에서 안전하게 관리합니다.

AI TOOL COMPARISON

Claude vs Google AI Studio — 개발 도구로서의 비교

경제숲을 만들면서 두 도구를 모두 사용해본 경험을 바탕으로 비교합니다.

C

Claude (Anthropic)

경제숲의 주력 개발 도구
✅ 장점
  • • 대화형 개발에 최적화 — 긴 맥락을 유지하며 코드 수정 가능
  • • 코드 품질이 매우 우수 — 복잡한 로직, 디버깅에 강함
  • • Artifacts — 코드를 대화 중 실시간 미리보기
  • • 프로젝트 기능 — 파일 업로드 + 맥락 기억
  • • Claude Code — 터미널에서 에이전트형 코딩
  • • 한국어 이해도 높음
⚠️ 단점
  • • Pro 월 $20 필요 (무료는 제한적)
  • • 상위 모델(Opus) 토큰 소진이 빠름
  • • 세션이 길어지면 맥락을 놓칠 수 있음
  • • 이미지 생성 불가
VS
G

Google AI Studio (Gemini)

보조 도구로 병행 사용
✅ 장점
  • • 넉넉한 무료 티어 — 부담 없이 시작 가능
  • • 토큰 소진율이 비교적 낮음
  • • Google 생태계 연동 (Sheets, Drive 등)
  • • 멀티모달 — 이미지, 영상, 오디오 입력 지원
  • • Gemini 2.5 Pro의 코드 품질 개선
  • • 긴 컨텍스트 윈도우
⚠️ 단점
  • • 대화형 개발 흐름이 Claude만큼 자연스럽지 않음
  • • Artifacts 같은 실시간 미리보기 부재
  • • 프로젝트/메모리 기능이 제한적
  • • 복잡한 코드 설계에서 Claude 대비 약함

결론: 하나만 고집하지 말 것

Claude는 "깊이 있는 개발 파트너", Google AI Studio는 "비용 효율적인 보조 도구"입니다. 프로젝트 단계, 작업 복잡도, 남은 토큰에 따라 유연하게 선택하는 것이 실전에서의 핵심 전략입니다. 결국 상위 모델을 쓰게 되며 토큰이 빠르게 소진되기 때문에, 병행 사용은 선택이 아닌 필수입니다.

COST CONTROL

DEV MODE — 비용 없이 개발하기

URL 뒤에 ?dev=true를 붙이면 AI 호출 없이 더미 데이터로 동작합니다.

🚨 왜 필요한가?

  • AI API는 호출할 때마다 비용 발생
  • 글자 색깔 하나 바꾸는 UI 작업에도 AI 호출하면 불필요한 비용
  • 실제 사고: 테스트 중 API 호출이 누적 → 하루 만에 $2.38 예상 외 비용
  • → 이후 DEV MODE 도입으로 개발 중 API 비용 완전 차단

✅ 어떻게 작동하는가?

  • DEV_DUMMY에 실제 API 응답과 동일한 형식의 더미 데이터
  • 빨간색 DEV 배너 + 워터마크 표시
  • 실제 출력과 100% 동일한 UI 확인 가능
  • 규칙: API 형식이 바뀌면 DEV_DUMMY도 반드시 함께 업데이트

Haiku 모델에서 뉴스 내용을 사실과 반대로 요약하는 문제가 실제로 발생했습니다. 해결: ① Haiku → Sonnet 업그레이드 ② 설명 앞 100자를 함께 전달 ③ 프롬프트에 검증 지시 강화.

인식 AI는 만능 비서가 아니라, 명령을 수행하는 로봇

나의 마음을 다 알아줄 것 같지만, 아직까지는 내가 정확한 프롬프트를 입력하고 역할을 부여해야 합니다. AI를 "다룰 줄 아는 능력"이 결과물의 품질을 결정합니다.

스킬 코딩을 몰라도 되지만, 알면 10배 빨라진다

글자 색깔 하나 바꾸는 것도, 코드를 모르면 AI에게 부탁하고 모든 프로세스를 거쳐야 합니다. 코딩 기초 지식이 있으면 간단한 수정은 직접 → 개발 속도가 비약적으로 빨라집니다.

기획 기획이 가장 중요하다

"일단 시작하고 고치면 되지" 접근은 위험합니다. AI가 코드를 한번에 다 생성하면 수정이 매우 어렵습니다. 처음부터 AI에게 말해야 합니다: "너는 나와 함께 방향을 잡아가며 앱을 만들 거야. 모든 부분 합의를 거쳐서 의사결정을 할 것."
